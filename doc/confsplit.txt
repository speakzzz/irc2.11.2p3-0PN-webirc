confsplit - SS/SU values in ircd.conf for irc2.11.2p3

######################################################################

This patch adds split-servers and split-users to ircd.conf to change
values without recompiling the ircd and to get around the volatile
nature of /SET SPLIT

Two new fields have been added to the M-line after the SID:

 M:<Server NAME>:<YOUR Internet IP#>:<Geographic Location>:<Port>
  :<SID>:<SS>:<SU>:

These values will be applied on server startup and (optionally) on
/REHASH. Note that they will still respect the minimum limits set in
config.h defines SPLIT_SERVERS and SPLIT_USERS (that is, they can
only override DEFAULT_SPLIT_SERVERS and DEFAULT_SPLIT_USERS).

example:

 M:irc.example.net:127.0.0.1:Example server:6667:0PNX:10:10000:

 Will set the split-servers (SS) current value to 10 servers minimum
 and split-users (SU) current value to 10000 users minimum, then do
 a check for split-mode.

Define CONFSPLIT in config.h to enable the patch. This will only
enable it on startup. To allow changing the values on a live server
with /REHASH you have to define CONFSPLIT_REHASH too.

Defining CONFSPLIT will add the character 'o' to the version string.
Defining CONFSPLIT_REHASH too will change that to an 'O'.
